# 🎓 LICT Campus Assistant — Hybrid AI Chatbot

A production-structured, full-stack **Streamlit** chatbot for Lumbini ICT Campus (LICTC).
It combines a **FAISS + Sentence-Transformers RAG pipeline**, the **Groq LLM API**,
**Tavily live web search**, secure **bcrypt-hashed accounts**, multi-session chat
history, 16 built-in visual themes, and multi-language UI support — merged here
from two prior prototypes into one clean, professional codebase.

## ✨ Features

- 🔐 **Accounts** — SQLite + bcrypt password hashing, brute-force lockout after repeated failed logins
- 🧠 **Hybrid RAG pipeline** — dense vector search (FAISS + Sentence-Transformers) over the campus knowledge base, with automatic fallback to Groq-hosted LLM reasoning and Tavily live web search
- 🎓 **Lumbini ICT Campus knowledge base** (`data/data.json`) — courses (BSc. CSIT, BCA, BIM, BHM), leadership, admissions, FAQs
- 🖼️ **Image search & safe URL fetching**, with SSRF-hardened link validation
- 🛡️ **Content moderation & input sanitization** on every message
- 🗂️ **Multi-session chat history** with auto-summarization for long conversations
- ⬇️ **Chat export** to Markdown and JSON
- 🌗 **16 built-in themes** (8 dark, 8 light) with a quick dark/light toggle
- 🌐 **Multi-language UI** (`src/ui/i18n.py`)
- ⚡ **Rate limiting** per session to control API spend and abuse
- 🕷️ **Optional data-ingestion tools** — a safe, allowlisted scraper and a TF-IDF chunk index for refreshing the knowledge base from the live campus website

## 🚀 Setup

1. **Install dependencies**

   ```bash
   pip install -r requirements.txt
   ```

2. **Run the app**

   ```bash
   streamlit run app.py
   ```

## 🎛️ UI Upgrades (Login + Agent Chat)

- Login screen is a single modern card with **animated starfield background** and **SVG icons** (no emoji).
- Chat screen includes the same starfield background.
- **Voice assistant (mic)**: click the mic button to dictate; speech-to-text fills the input.
- **File upload (chat + ask combined)**: attach `.txt` or `.md` files; the content is appended into the prompt under **[Attached file content]**.

## 🗄️ Database

On first run, `src/core/database.py` auto-creates/migrates `data/chatbot.db` (SQLite) with:

- `users` — accounts, bcrypt password hash, lockout tracking
- `conversations` — one row per chat session
- `messages` — every message, with source tag (Knowledge Base / Web Search / Image Search / etc.)
- `conversation_summaries` — rolling auto-summaries for long chats

## 🧠 How answers are generated

1. The query is sanitized and moderated (`src/core/rag.py`).
2. It's embedded and matched against the campus knowledge base via FAISS.
3. If the match is strong, the answer is grounded in that knowledge-base entry.
4. Otherwise, the pipeline falls back to the Groq-hosted LLM, optionally blended
   with a live Tavily web search for current information.

## 🕷️ Refreshing the knowledge base (optional)

```bash
python -m src.tools.scraper        # re-crawl lict.edu.np into data/data.json
python -m src.tools.search_index   # rebuild data/chunks.json
# or:
bash scripts/rebuild_search_index.sh
```

The scraper only ever fetches `lict.edu.np` (allowlisted host), strips scripts/styles,
caps total pages, and times out safely on failures.

## 🔒 Security notes

This is a learning/demo project. For production you'd additionally want to:

- Serve behind HTTPS with a reverse proxy
- Add email verification for registration
- Add CSRF protection if exposed publicly
- Move secrets to a managed secrets store instead of `.env`
